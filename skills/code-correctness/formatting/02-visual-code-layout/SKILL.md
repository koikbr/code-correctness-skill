## Transcript
This is a well-written code. The names reveal intent. The methods are focused. Yet it's still hard to read. All because there's no vertical spacing. As soon as we add that spacing, every blank line tells you: new concept starts here. You scan instead of reading every line. Navigate to any section in a glance. The mental effort drops because your brain processes groups far more easily than a wall of text. But spacing works both ways. If blank lines separate concepts, then related lines should stay vertically dense. Take this block of code. Spacing everywhere disconnects related code. Spacing nowhere makes it all look the same. Both are equally hard to scan. Instead, let density show what belongs together. Data extraction at the top, business logic in the middle, and the result at the end. 3 distinct groups, each with a single clear purpose. With our blocks defined, there's just one more layer of structure to add: proper indentation. Where each scope level gets its own indent, and your code becomes a visual hierarchy you can navigate at a glance. Good formatting means your code speaks before it's read. Follow for more clean code principles coming next.

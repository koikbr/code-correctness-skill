## Transcript
Look at this. Zero arguments. There's nothing to remember, nothing to mess up. This right here is the ideal function call. Add two arguments and things change. Order starts to matter and your test cases multiply to cover the combinations. At five arguments, you've entered chaos. Test cases explode with every addition and each parameter is one more thing developers need to remember when they use your function. Why does this matter? Because arguments carry mental weight. More arguments lead to more confusion, more bugs, and more trips back to the documentation. Here's how to fix it. When arguments pile up, they're actually telling you something. They want to become an object. Group the related ones together, give that grouping a name, and pass the idea instead of the pieces. So remember, zero arguments is ideal. 1 is clear, 2 is acceptable, but 3? That's your signal to wrap them up. Follow for more clean code principles coming next.

## Transcript
Don't delete that code. What if we need it again? So you comment it out. It starts with one function here, a few lines there, maybe an entire block somewhere else. But now your file is half alive, half dead, like artifacts in a museum. So stop keeping it around. Delete the commented code. All of it. If you need it back, your version control has every line you've ever written. This also means you don't need attributions in your code. Having the author's name at the top seems useful. If you have a question, you know who to talk to. But over time, other people make changes, the file evolves, and that name at the top? It stays there forever, long after it's stopped being relevant. The same goes for journal comments, those dated logs of every edit. Your version control already does that too. Better than any comment ever could. So please leave the history to git. Your code should only contain what runs today. Everything else is already in your history. Follow for more clean code principles coming next.

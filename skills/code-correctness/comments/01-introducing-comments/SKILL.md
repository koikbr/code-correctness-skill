## Transcript
Comments lie. Not intentionally, but they do. Here's how. Code changes and evolves. It moves, it splits, it merges. But comments? They get left behind, still telling the old story while the code writes a new one. An inaccurate comment is worse than no comment at all. It doesn't just fail to help, it actively deceives. So, should you never write comments? No. But you need to know which ones help and which ones hurt. That's what's coming next. Follow for the clean code principles coming next.

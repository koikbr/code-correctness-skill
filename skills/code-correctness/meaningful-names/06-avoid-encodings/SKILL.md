## Transcript
Believe it or not, in 1999, this naming style was considered best practice. But today, it's a code smell. We call this Hungarian notation, encoding the type into every variable name. You see, back then, editors couldn't tell you the type. You actually had to remember it yourself. But here's the thing, your IDE already knows. So really, why encode it twice? Clean Code calls this principle avoid encodings. Just watch, modern IDEs show you the type on hover. Plus, compilers catch type errors instantly. Basically, the encoding solved a problem that no longer exists. So go ahead and strip them away. Let your tools handle the types and let your names describe the intent. Follow to write cleaner code.
